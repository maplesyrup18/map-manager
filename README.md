# Welcome to the workshop: Vibe Coding & AI Driven Development

You're now in a cloud-based development environment where you can create applications using **Kiro** - an agentic AI tool that enables development from prototype to production, understanding natural language and writing code for you.

## Getting Started
All the info you need is in the workshop guide. Use the guidelines to create a working application by simply describing what you want in plain English. No coding experience required!

## Step 1: Login to Kiro CLI (Terminal)

1. **Open a terminal** by clicking **Terminal** in the top menu, then **New Terminal**
2. **Type this command** and press Enter:
   ```
   kiro-cli login --use-device-flow
   ```
3. **Select "Use with Builder ID"** when prompted
4. **Follow the device flow authentication process**:
   - Copy the device code shown in the terminal
   - Open the provided URL in your browser (on your local machine)
   - Enter the device code and complete authentication
5. **Return to the terminal** - you should see a success message

## Step 2: Test Kiro CLI

1. **In the terminal, type**:
   ```
   kiro-cli
   ```
2. **When the Kiro prompt appears, type your question**:
   ```
   What is vibe coding, and why is it interesting for me?
   ```
3. **Important**: If Kiro asks for web search permission, **type `t`** to trust and allow access to current information
4. **Press Enter** and see Kiro's response

## Step 3: Amazon Q Developer (VS Code Extension) - Optional

You can use this workshop with just Kiro CLI from the terminal. However, if you'd like to also use the Amazon Q Developer extension in VS Code:

1. **Find the Q icon** in the left sidebar (it looks like a "Q")
2. **Click the Q icon** to open the Amazon Q panel
3. **Follow the instructions in your browser** to authenticate with Builder ID
4. **Return to VS Code** - you should now see "Amazon Q" is connected

**Alternative**: If you prefer to focus only on Kiro CLI, simply click the **File Explorer** icon in the left sidebar to hide the Amazon Q panel and work directly with files and the terminal.

## Step 4: Open Your Workspace

If you don't see files in the left sidebar:
1. **Click File** in the top menu
2. **Click "Open Folder"**
3. **Navigate to** `/home/ec2-user/dev-server`
4. **Click OK**
